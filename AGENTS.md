# Project Instructions

- Use `frontend-developer` for UI-only work: HTML, Tailwind, responsive behavior, accessibility, interaction design, and frontend performance.
- Use `fullstack-ui-ux-engineer` when a task also involves API contracts, persistent data, authentication, server state, or backend architecture.
- Use both skills for complete product features spanning UI and data or backend work.
- Preserve the visual distinction between the three portal concepts unless a task explicitly changes a concept's direction.
